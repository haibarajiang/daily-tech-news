    随着闭源模型不断刷新能力上限，开源与闭源大模型之间的差距再次成为焦点。闭源模型在算力与数据护城河下保持领先，而开源模型则在定制化和成本上发力。这一差距的演变将深刻影响未来 AI 产业的权力分配与开发者的技术选型。

- [AI 智能体的身份与权限挑战](https://www.infoq.cn/article/tDY9pS7LlrVIBq1pPotd?utm_source=rss&utm_medium=article) (InfoQ)
    随着 Agent 成为云服务新核心用户，Uber 和 Auth0 正在重新思考访问控制。当 AI 代理能够自主调用 API 和数据时，传统的基于人的身份验证体系面临失效风险。为 Agent 建立独立的身份与最小权限机制，已成为 AI 基础设施安全的关键命题。

## 3. 工具与深读

- [Show HN: Smart model routing directly in Claude, Codex and Cursor](https://github.com/workweave/router) (Hacker News)
    Workweave Router 是一款专为编程 Agent 设计的智能路由插件，支持 Claude Code、Codex 和 Cursor。它能根据请求内容自动将任务分配给最合适的模型，帮助开发者在不同模型间平衡性能与成本，是应对多模型时代的实用提效工具。

- [Show HN: DBOSify – Drop-in Temporal replacement built on Postgres](https://github.com/dbos-inc/dbosify-py) (Hacker News)
    DBOSify 是一个基于 Postgres 构建的 Temporal 替代品，提供无缝替换体验。对于需要分布式工作流编排但又不想维护复杂独立基础设施的开发者来说，利用现有的 Postgres 数据库即可实现可靠的微服务编排，大幅降低了系统复杂度。

- [MicroVMs: Run isolated sandboxes with full lifecycle control](https://aws.amazon.com/blogs/aws/run-isolated-sandboxes-with-full-lifecycle-control-aws-lambda-introduces-microvms/) (Hacker News)
    AWS Lambda 引入 MicroVMs，提供具有全生命周期控制的隔离沙箱环境。这一更新增强了函数计算的安全隔离性与灵活性，对于需要运行不可信代码或高安全边界要求的 AI 代码执行场景，提供了更底层的基础设施支持。

- [API7.ai创始人：烧了几百亿Token，用AI重写生产级网关的6条经验](https://www.infoq.cn/article/5YIK6SvTf5h07YckXZd3?utm_source=rss&utm_medium=article) (InfoQ)
    API7.ai 创始人分享了使用 AI 重写生产级网关的实战经验。文章详细总结了烧掉几百亿 Token 后得出的 6 条核心避坑指南，对于想要在核心业务代码中引入 AI 辅助编程的架构师和开发者而言，是极具参考价值的深度实践复盘。

- [A C++ implementation of a fast hash map and hash set using hopscotch hashing](https://github.com/Tessil/hopscotch-map) (Hacker News)
    Tessil 开源的 hopscotch-map 是一个使用跳房子哈希算法的 C++ 高性能哈希映射和集合实现。相比传统哈希表，它在缓存友好性和最坏情况延迟上表现更优，适合对性能极其敏感的 C++ 底层开发与高频交易等场景。
